# Darwin Live Anchor Script

Target length: 15-30 seconds.

## Opening

Darwin evolves retrieval strategies instead of hardcoding one RAG pipeline. Each genome controls retrieval, coordination, and durability choices. We run the same query through competing genomes, judge the answers, and use that score as fitness for the next generation.

## During The Demo

Here, the population is moving through generations. The fitness curve shows answer quality improving. The champion panel shows which genome is currently winning and which retrieval choices changed from the baseline.

## Closing

The important part is that Darwin is not only producing an answer. It is learning which retrieval strategy produced the best answer, then using that signal to evolve the system.

## If Time Is Very Short

Darwin is an evolutionary RAG system. Genomes compete, answers are scored as fitness, and better retrieval strategies survive into future generations.
